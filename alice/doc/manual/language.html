Alice ML is based on the Standard ML programming language (SML), as defined in

  • Robin Milner, Mads Tofte, Robert Harper, Dave MacQueen:
    The Definition of Standard ML (Revised)
    The MIT Press
    ISBN 0-262-63181-4

However, Alice ML features a number of extensions relative to SML (including most current proposals for Successor ML).

  • Futures:
    • concurrency
    • lazy evaluation
    • promises
  • Extended type language:
    • structural datatypes
    • extensible types
    • type wildcards
  • Higher-order modules:
    • local modules
    • generalised open declarations
    • higher-order functors
    • signature members and abstract signatures
    • lazy evaluation
    • fixity specifications
    • syntax enhancements
  • Packages:
    • first-class modules
    • dynamic typing
  • Pickling:
    • persistent values
    • persistent modules
  • Components:
    • separate compilation
    • lazy linking
    • first-class components
    • sandboxing
  • Distributed programming:
    • tickets
    • proxies
  • Constraint programming
    • finite domain constraints
    • finite set contraints
    • first-class computation spaces
  • Syntactic sugar:
    • line comments
    • extended syntax for numeric literals
    • vector expressions and patterns
    • sugar for records
    • extended pattern language
    • generalized recursive definitions
    • Optional bars and semicolons
    • conditional with optional else branch
    • finalization
    • source position
    • assertions
    • statements

You should also be aware of some

  • incompatibilities with SML,
  • limitations of the current implementation.
  • The comment marker (*) introduces a line comment.
  • Numeric literals may contain underscores to group digits.
  • The prefixes 0b and 0wb introduce binary literals.

Alice ML defines some additional identifiers as reserved words:

	any    assert    assertd   comp      constructor  exttype
	fct    finally   from      import    non          lazy
	pack   spawn     unpack    withfun   withval      _file_    _line_
